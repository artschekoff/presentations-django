# Новые требования к именованию файлов и форматам

Документ описывает новые именования к форматам и названиям сохраняемых файлов

## Требования

• оставляем только форматы pptx + text (pdf убираем)
• имя папки = task_id
• имя файла презентации = task_id
• имя файла текста = task_id
• генерируем по 20 слайдов на презентацию
• скриншоты не должны сохраняться

## Уточнения

| Вопрос | Ответ |
| ------ | ----- |
| Что использовать если `task_id` отсутствует? | UUID презентации (`Presentation.id`) |
| Как фиксировать 20 слайдов? | Значение по умолчанию = 20, можно переопределить через запрос |
| Что делать с кодом скриншотов? | Оставить env-переменную `SAVE_SCREENSHOTS=false` |
| `presentations_module` сама именует файлы — как реализовать? | Переименовываем папку и файлы после завершения генерации |

## Критерии приемки

При обработке задачи выполняются требования:

- Папка называется `task_id`; если `task_id` не передан — UUID презентации
- Файлы `<task_id>.pptx` и `<task_id>.txt` лежат внутри этой папки
- PDF-файл не создаётся
- `slides_amount` по умолчанию = 20 (можно переопределить в запросе)
- Скриншоты не сохраняются
